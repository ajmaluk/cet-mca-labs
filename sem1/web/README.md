# Web Programming Lab

This directory contains the laboratory experiments for **Web Programming** as part of the **CET MCA Semester 1** curriculum. The labs focus on building modern, responsive, and interactive web applications using HTML5, CSS3, JavaScript, and PHP.

## 🚀 Lab Overview

### Phase 1: Static Web Technologies (HTML & CSS)
Fundamental web structure and styling using modern design principles like **Glassmorphism**.
- **Experiment 1**: [Web Browsers & Servers](./program1.html)
- **Experiment 2**: [HTML Multimedia](./program2.html)
- **Experiment 3**: [Internal & External Linking](./program3.html)
- **Experiment 4**: [Frames & Layouts](./program4.html)
- **Experiment 5**: [HTML Lists](./program5.html)
- **Experiment 6**: [Tables & Cell Spacing](./program6.html)
- **Experiment 7**: [Registration Form Design](./program7.html)

### Phase 2: Dynamic Styling & Interactivity (JS)
Enhancing user experience with CSS3 animations and JavaScript validation.
- **Experiment 8**: [CSS3 Styling & Backgrounds](./program8.html)
- **Experiment 9**: [JavaScript Form Validation](./program9.html)
- **Experiment 10**: [Event Handling & Callbacks](./program10.html)

### Phase 3: Server-Side Programming (PHP & MySQL)
Building full-stack applications with database integration.
- **Project 1**: [Bill Generation System](./php/generate-bill.php)
- **Project 2**: [Student Registration & Database Analysis](./php/student-reg.php)

## 🛠 Features
- **Glassmorphic UI**: Consistent modern aesthetic across all labs.
- **Responsive Design**: Mobile-first approach using CSS Flexbox and Grid.
- **SEO Optimized**: Each page includes relevant meta tags for "CET MCA" visibility.
- **Clean Code**: Semantic HTML5 and modular CSS variables.

## 📚 Syllabus Alignment
These programs are designed based on the **Lab-MCA105: Web Programming Lab** syllabus for students at the **College of Engineering Trivandrum**.


## 📝 Notes & Best Practices
- Use semantic HTML tags (header, nav, main, section, footer) for structure and accessibility.
- Add alt text to all images for accessibility.
- Validate your HTML and CSS using online validators.
- For PHP/MySQL, always check database connection and query results.
- Use comments in your code to explain complex logic or important sections.
- Test your web pages in multiple browsers for compatibility.
