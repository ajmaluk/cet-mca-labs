Client/Server Model

The client–server model is a distributed application structure that partitions tasks or workloads between the providers of a resource or service, called servers, and service requesters, called clients. Clients initiate communication sessions with servers, which await incoming requests. Common examples include email, web browsing, and network printing.

Components of Web Apps

A modern web application consists of several layers: the client-side (frontend), the server-side (backend), and the database. Every time a user sends a request, the server processes it, interacts with the database if necessary, and sends the response back to the browser.

Web Content Types

Web content includes any writing and digital media published on the internet. Major formats include:

  • Text Content: Articles, blogs, and documentation.
  • Visual Content: Photography, infographics, and illustrations.
  • Multimedia: Audio recordings, podcasts, and streaming video.

HTTP (Hypertext Transfer Protocol)

HTTP is the foundation of data exchange on the web. It is a request-response protocol where a client makes a request and a server provides a response.

Common HTTP Methods:

  • GET: Used to request data from a specified resource.
  • POST: Used to send data to a server to create/update a resource.
  • PUT: Used to update an existing resource.
  • DELETE: Used to remove a specific resource.

Dynamic Web Pages

Unlike static pages that show fixed content, dynamic pages change based on user interactions, user profiles, or real-time data from a database. They enable personalization and interactive features like shopping carts and social feeds.

Application Servers

An application server handles the business logic of an application. It sits between the web server and the database, managing resource allocation, security, and complex data processing.

Web Security

Web security involves protecting applications from unauthorized access and attacks. This includes implementing encryptions (SSL/TLS), firewalls, and secure coding practices to prevent vulnerabilities like SQL injection and Cross-Site Scripting (XSS).
